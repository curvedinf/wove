# 2.0.0

`2.0.0` is the first release in Wove's remote task environment series. It keeps Wove's inline workflow model, but adds the execution-environment layer needed to send selected tasks outside the current Python process.

- Adds project configuration through `wove.config(...)` and optional `wove_config.py` discovery.
- Adds named environments so a project can keep most work local while routing selected tasks elsewhere.
- Adds the executor interface for local execution, stream-based subprocess execution, network executor delivery, and adapter-backed remote task execution.
- Adds HTTP/HTTPS, gRPC, and WebSocket network executors for direct worker-service integrations.
- Adds remote task environments for existing queues, workflow engines, clusters, and schedulers through built-in adapters.
- Adds callback-based result delivery for remote task execution, including normalized delivery errors, timeouts, cancellation policy, heartbeat handling, and orphan handling.
- Moves dispatch serialization behind the optional `wove[dispatch]` extra so local Wove keeps a minimal dependency footprint.
- Keeps backend libraries optional: adapter code ships with Wove, but each backend package is required only when that adapter is selected.
- Adds the Sphinx and Shibuya documentation site, including topic guides, reference pages, and adapter setup pages.
